## Quick Reference: How to Request a Report

### Step 1: Identify What You Need
Choose from our available reports:

| Report Type | Best For | Typical Turnaround |
|-------------|----------|-------------------|
| **Student Profile** | Parent meetings, counseling sessions | Same day |
| **Attendance by Date** | Daily roll call, absence follow-up | Within 1 hour |
| **Enrollment Summary** | Class planning, capacity analysis | Same day |
| **Custom Data Export** | SEQTA sync, Power BI updates | 1-2 business days |

### Step 2: Submit Your Request
**Email:** ict@stc.edu.au  
**Subject Line:** `[DATA REQUEST] - [Report Type] - [Your Name]`

**Example:** `[DATA REQUEST] - Student Profile - Sarah Johnson`

## Understanding Your Report Results

### Student Profile Report
**What You'll Receive:** 3 sections

#### Section 1: Student Overview
| Column | What It Means |
|--------|---------------|
| **Student Number** | Unique identifier (matches SEQTA) |
| **Full Name** | First + Last name |
| **Year Level** | Current grade (7-12) |
| **Contact Info** | Parent/guardian phone and email |
| **Total Enrollments** | Number of classes this term |
| **Attendance Rate** | Percentage of days present (target: >90%) |

#### Section 2: Current Classes
| Column | What It Means |
|--------|---------------|
| **Class Name** | Subject and section (e.g., "Math 8A") |
| **Subject** | Core subject area |
| **Teacher** | Assigned staff member |
| **Current Grade** | Latest assessment result |
| **Status** | Active / Withdrawn / Completed |

#### Section 3: Recent Attendance
| Column | What It Means |
|--------|---------------|
| **Date** | Attendance date |
| **Class** | Which class this attendance record is for |
| **Status** | Present / Absent / Late / Excused |

### Attendance Report
| Column | What It Means |
|--------|---------------|
| **Student Name** | Full name |
| **Class** | Subject and section |
| **Teacher** | Staff member who marked attendance |
| **Status** | Present / Absent / Late / Excused |
| **Attendance Rate** | Daily percentage (shown in summary) |

**üö® Red Flags to Watch For:**
- Attendance rate <85% (chronic absenteeism)
- Multiple "Absent" without "Excused" (unexplained absences)
- Late arrivals >3 times per week (pattern concern)

### Enrollment Summary Report
| Column | What It Means |
|--------|---------------|
| **Class Name** | Subject and section |
| **Current Enrollment** | Number of students enrolled |
| **Max Capacity** | Maximum class size |
| **Utilization %** | How full the class is |
| **Status** | Full / Near Capacity / Available / Under-enrolled |

**üìà Capacity Status Guide:**
- **Full (100%):** No seats available
- **Near Capacity (90-99%):** Limited seats
- **Available (50-89%):** Seats available
- **Under-enrolled (<50%):** Consider consolidation

## What We Can and Can't Do (Privacy & Confidentiality)

### ‚úÖ What We CAN Provide
- **Individual student data** for students in your classes or under your care
- **Aggregated class data** for planning and analysis
- **Attendance summaries** for roll call and follow-up
- **Academic performance data** for students you teach or counsel
- **Enrollment statistics** for capacity planning

### ‚ùå What We CANNOT Provide
- **Student data for students not in your classes** (unless you're leadership/counselor)
- **Bulk student contact information** for marketing purposes
- **Sensitive medical or behavioral notes** (stored in separate system)
- **Data for external parties** without proper authorization
- **Historical data beyond 2 years** (archived separately)

### üõ°Ô∏è Child Safety Compliance
**All data requests are logged and audited.**

- You must have a legitimate educational purpose for requesting data
- Student data cannot be shared outside StC without parent consent
- Screenshots or printed reports must be stored securely
- Data must be deleted/destroyed when no longer needed
- Report any suspected data breaches immediately to ICT

**If you're unsure whether you should have access to specific data, ask first.**

## Tips for Better Reports

### Do's ‚úÖ
- **Be specific:** "Attendance for Year 8 on Jan 15" beats "Need attendance data"
- **Provide context:** Helps us prioritize and format appropriately
- **Use student numbers:** Faster lookup than names (especially for common names)
- **Plan ahead:** Request reports 24 hours before meetings when possible
- **Follow up:** If you don't hear back in expected timeframe, send a reminder

### Don'ts ‚ùå
- **Don't request data you don't need:** Respect student privacy
- **Don't share login credentials:** Each staff member should have their own access
- **Don't export to personal devices:** Use school-issued equipment only
- **Don't modify data yourself:** Report errors to ICT for correction
- **Don't assume data is real-time:** Reports reflect last database refresh (typically 4-hour lag for Power BI)

## How to Report Issues

### Data Looks Wrong?
**Email ICT with:**
1. Report type and date generated
2. Specific discrepancy (e.g., "Student shows absent but was in my class")
3. Expected vs. actual values
4. Screenshot if helpful

**Example:**
```
Subject: [DATA ISSUE] - Attendance Discrepancy - Jan 15

Hi ICT Team,

The attendance report for Year 8 on Jan 15 shows:
- Student: John Smith (12345)
- Status: Absent
- Class: Math 8A

However, John was present in my class that day. Can you please verify?

Thanks,
Sarah Johnson
```

### Need Help Interpreting Results?
- Schedule a 15-minute training session with ICT
- We can walk through report columns and what they mean
- Available during planning periods or after school
